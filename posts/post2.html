Understanding DOM Ready: jQuery vs Vanilla JavaScript

This post explains how to execute code when the DOM is fully loaded using two methods: jQuery’s $(document).ready() and the native JavaScript document.addEventListener('DOMContentLoaded', ...). Below you’ll find details on what each does, when they fire, their syntax, and key features.

1. jQuery: $(document).ready()

What it does: This jQuery method executes a callback function once the DOM is fully loaded and ready to be manipulated. It waits for the HTML to be parsed and the DOM tree to be constructed, but it does not wait for external resources like images, stylesheets, or scripts.

Timing: It fires when the DOM is "interactive," typically before the DOMContentLoaded event in some edge cases because jQuery optimizes for usability and may trigger slightly earlier.

Syntax

$(document).ready(function() {
        console.log("DOM is ready via jQuery");
    });

Key Features

  • jQuery-specific: Requires the jQuery library.
  • Can be called multiple times; all callbacks will execute in the order they were added.
  • Ignores whether the event has already fired (if added late, it still runs the callback immediately).

2. Vanilla JavaScript: document.addEventListener('DOMContentLoaded', ...)

What it does: This native JavaScript event listener triggers when the DOM is fully loaded and parsed, similar to $(document).ready(). It is part of the standard DOM API and does not rely on external libraries.

Timing: It fires when the DOMContentLoaded event occurs, which is after the HTML is fully parsed but before external resources (like images) are loaded. It is tied directly to the browser’s event lifecycle.

Syntax

document.addEventListener('DOMContentLoaded', function() {
        console.log("DOM is ready via DOMContentLoaded");
    });

Key Features

  • Native JavaScript: No dependencies required.
  • Only runs if added before the DOMContentLoaded event fires; if added after, it won’t execute unless the event hasn’t occurred yet.
  • Can only have one listener per event type unless you add more using addEventListener.
